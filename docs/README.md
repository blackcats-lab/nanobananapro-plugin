## 読む順番

初めてプロジェクトに参加する場合は、以下の順序で読むことを推奨します。

1. **基本設計書** - プロジェクト全体像の把握
2. **機能設計書** - 各機能の仕様理解
3. **API 設計書** - 外部 API 連携の詳細
4. **詳細設計書** - 実装レベルの理解
5. **テスト設計書** - テスト方針とケースの確認
